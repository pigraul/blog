## snpEff权限问题

由于container的权限是user，导致snpEff在container中运行数据库下载时，没有写入权限。解决方法是给`-dataDir`指定一个目录。

但是在nextflow中，如果给定一个相对路径，例如`-dataDir ./data`，snpEff仍会报错，因为Docker容器内的当前目录往往和宿主机的预期不符，且Java程序(SnpEff)在解析相对路径时，经常会回退到其安装目录（例如`/opt/conda/...`），所以需要将路径转换为绝对路径，并确保SnpEff能够识别。

```snp.nf
// 在 script 块外部或内部定义基础逻辑
def default_cache = "\$(pwd)/snpeff_data"
def snpeff_cache = params.snpeff_cache ?: default_cache

script:
"""
if [ "${snpeff_cache}" = "${default_cache}" ]; then
    mkdir -p ${snpeff_cache}
fi
celescope snp analysis_snp \\
    --sample ${sample} \\
    --snpeff_cache ${snpeff_cache} \\
    ...
"""
```
